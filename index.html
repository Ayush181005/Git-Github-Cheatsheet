Git and Git Hub Cheatsheet
  1. Configuring name and email in Git
    2. git config --global user.name <Your Name>
    git config --global user.email <Your Email>
    To Check Whether it is set or not:- 
    git config --global user.name
    git config --global user.email


  2. Initializing Empty Git Repository
    3. git init
    it creates a folder, you can check it by 
    ls -lart


  3. Checking the status
    4. git status

    To see a short status:- 
    git status -s

    M <filename> //Modified in Staging area
    M <filename> //Modified in Working Tree
    M M <filename> //Modified in both Working Tree and Staging area


  4. To take the file from untracked to Staging:-
    5. git add <filename>
    To add all files at once: 
    git add -A
    Now, the file has come on the staging stage

  5. Initial Commit:-
    6. git commit -m "<Your Commit Message>"

    Another way through VIM editor:-

    git commit
    To write anything (commit message), press:
    I
    Write the commit message there on the top 
    Initial commit
    To exit the editor, press these keys one by one:
    Esc, :, w, q, Enter


  6. To match a file or many files with the working tree with the last commit:-
    7. git checkout <filename>
    To do this for all files at once:- 
    git checkout -f


  7. To see the commit logs by different authors:-
    8. git log
    Filtering outputs of commit log:- 
    git log -p -<no. of commits>
    By doing this, it will also show all the changes Press 'Q' to quit

  8. Comparing and seeing Difference:-

    • Between the staging area and working tree:-
      • git diff
    • Between the staging area and last commit:-
      • git diff --staged


  9. Directly Commiting from modified file:-
    10. git commit -a -m "<Your Commit Message>"


  10. Removing files:-

    • From working tree and staging area:-
      • git rm <filename>
    • Only from the staging area:-
      • git rm --cached <filename>
    After deleting file, NOTE to commit also to make the status normal

  11. Ignoring files:-

    12. Make a file from GIT Bash only as in windows it is not possible directly 
    touch .gitignore
    In .gitignore file, write the files/extensions of files/file path/directory to ignore:-

    • foldername/ - This will do for this folder
    • filename - This will do for all files of this name
    • *.extension - This will do for all files of this extension
    • foldername/filename - This will do for the file in the folder
    • /filename - This will do it for the file in the root folder


  12. Branches:-

    • Seeing how many branches are there:-

      • git branch
    • Creating new branch:-

      • git branch <branchname>
    • Switching branches:-

      • git checkout <branchname>
    • Merging branches:-

      • git checkout <master>
      git merge <branchname>
    • Creating new branch and switching to it:-

      • git checkout -b <branchname>


  13. Remote repository:-

    • Adding this to remote repository:-
      • git remote add origin <repository-URL or SSH>
    • Checking remotes:-
      • git remote
    • Getting fetch and push URLs:-
      • git remote -v


  14. Pushing to remote repository:-
      15. git push <remote(origin)> <branchname>
      To push according to the -u command:- 
      git push
      TO push every time to a branch every time through 'git push' command:- 
      git push -u <branchname>


  15. Pulling:-
      16. pulling all the changes done by anyone in the repository 
      git pull <remote(origin)> <branchname>


  16. Checking for changes done by us:-
    • for changes in MODIFIED file:-
      • Getting changes in all files:- 
      git diff
      Getting changes for a file:- 
      git diff <filename>
    • For changes in STAGED area:-
      • Getting changes in all files:- 
      git diff --staged
      Getting changes for a file:- 
      git diff --staged <filename>
    NOTE: write filename along with the extension

  17. Unstaging:-
      18. git reset <filename>
      For changing the file, 
      git checkout <filename>


  18. Adding Computer Access:-

    • Go to 'Settings'
    • Go to 'SSH and GPG keys'
    • Click on 'generating SSH keys'
    • Click on 'Generating a new SSH key and adding it to the ssh-agent'
    • Do the steps given on the page


  19. To change URL:-

    20. git remote set-url origin <URL(SSH or HTTPS or etc.)>


  20. Cloning (It is possible only for public repos):-

    21. Open GIT Bash in some another folder 
    git clone <URL of Repo> <foldername>
    Without the foldername, it will clone in a folder with a name of the repo
